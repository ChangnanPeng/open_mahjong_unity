# 简单麻将规则定义

## 牌张与基本流程

- 四人麻将，使用万、筒、索和七种字牌，共 136 张。
- 无花牌、无王牌区、无定缺、无查叫、无错和流程。
- 合法牌型包括标准四面子一雀头、七对和十三幺。
- 合法牌型即使没有任何计分番种，也允许以 0 分和牌。
- 吃只允许出牌者的固定下家；碰和杠按通常优先级处理。
- 吃、碰后必须出牌；明杠、暗杠、加杠后补牌并重新检查手牌动作。
- 最后一张弃牌只允许和牌响应，不允许吃、碰、杠。

## 终局流程

- 当前版本固定采用“一人和牌即止”。
- 自摸确认后立即结束本局。
- 一张弃牌或一张加杠牌有多名玩家可和时，按责任玩家之后的固定座次头跳，最近的首个合法赢家结束本局。
- 首个赢家确认后不继续处理其余和牌响应，也不进入下一巡。

## 过和限制

- 放弃某张弃牌的和牌机会后，在自己下次出牌前不能再点和同一牌。
- 玩家出牌时，原限制集合被替换为自己刚打出的牌。
- 自摸不受此限制；抢加杠和按点和处理。

## 庄家与局数

每局固定轮庄，不因和牌者、点炮者或流局改变。达到配置局数后结束整场。

## 计分支付

设和牌番值为 `X`，合计最高按 20 番计算：

- 点和或抢杠和：责任玩家支付 `6X`。
- 自摸：其余三家各支付 `2X`，赢家共获得 `6X`。
- 0 番合法和牌不产生分数转移，但仍立即结束本局。

## 信息与提示

- 暗杠者看到真实牌，其他玩家只收到隐藏后的暗杠数据。
- 服务端在最终结算中公开赢家手牌、番种和分数变化。
- 客户端本地听牌、悬停和回放铳牌提示仅用于交互；局面番与最终和牌结算以服务端为准。
